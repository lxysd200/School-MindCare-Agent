# 开发日志（Devlog）

目标：用最小的维护成本，长期记录 MindBridge 从原型到完整系统（Agent / Backend / RAG / MCP）的演进过程。

## 记录规则

- 每次完成一个可描述的闭环就追加一条：做了什么、为什么这么做、如何验证
- 尽量把“设计动机”和“踩坑结论”写清楚，细节代码留在 commit 里
- 重要里程碑在 GitHub 打 tag（例如 `v0.1-agent-core`）

## 日志模板

### YYYY-MM-DD

- 目标：
- 完成：
- 关键实现：
- 遇到的问题：
- 结论 / 下一步：
